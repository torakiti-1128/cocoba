# 状態遷移

## 1. 状態遷移の基本原則
本システムは、エッジPC上で稼働するステートマシンによって一元管理される。
* **排他制御:** システムは常に単一の「状態（State）」にのみ存在し、複数のアクションを同時進行させない。
* **フェイルセーフ優先:** タイムアウトや異常を検知した場合は、いかなる状態からでも即座に「停止・待機」へ強制遷移させ、ハードウェアの暴走を防ぐ。
* **非同期介入の許可:** Webシステムからの「緊急停止（キルスイッチ）」は、すべての状態遷移を最優先で上書きする。

## 2. 正常系フロー (Happy Path)

1. **[STATE 0: 待機]**
   * **動作:** カメラ映像を常時取得し、YOLOv8nで空間をスキャン。
   * **Web同期:** 「待機中」ステータスを一定間隔でクラウドへ送信。
   * **遷移条件:** 「排泄ポーズ」または「排泄物」を信頼度設定値以上で検知した場合、[STATE 1]へ遷移。

2. **[STATE 1: 検知]**
   * **動作:** 排泄物の絶対座標(X, Y)を確定し、ターゲットとしてメモリに保持。証拠画像（スナップショット）を切り出す。
   * **Web同期:** 証拠画像をクラウドへアップロードして「検知」ステータスを同期。
   * **遷移条件:** 座標確定後、直ちに[STATE 2]へ遷移。

3. **[STATE 2: 誘導]**
   * **動作:** おやつシューター（ESP32）へ「排出イベント」を送信。犬を排泄地点から物理的に引き離す。
   * **遷移条件:** カメラ映像内でわんちゃんが排泄物の座標から規定距離（例: 1.5m）以上離れたことを確認できた場合、[STATE 3]へ遷移。

4. **[STATE 3: 出動]**
   * **動作:** Cocobaロボットへ出動イベントを送信。ArUcoマーカーを追跡し、ビジュアルサーボ制御により目標座標へ移動させる。
   * **Web同期:** 「出動中」ステータスへ更新。
   * **遷移条件:** ロボットの現在座標と目標座標の誤差が許容範囲内（例: 半径5cm以内）に収束した場合、[STATE 4]へ遷移。

5. **[STATE 4: 投下 / Deploying]**
   * **動作:** 投下機構のサーボモーターを駆動し、ドームを1つ落下させる。
   * **Web同期:** 投下完了のステータス更新と、飼い主のスマホへ「シールド完了」のプッシュ通知を送信。ドーム残弾数を-1する。
   * **遷移条件:** モーターの駆動完了後、[STATE 5]へ遷移。

6. **[STATE 5: 帰還]**
   * **動作:** ロボットを初期位置（待機場所/充電ドック）へ後退・旋回させて戻す。
   * **遷移条件:** 初期位置への到着を確認後、[STATE 0]へ戻る（複数検知がある場合は、残弾があれば再度[STATE 3]へ）。

## 3. 異常系フローとフェイルセーフ
現場で起こりうるトラブルに対するシステムの自己防衛ロジック。これらが発生した場合、システムは即座に安全状態へ移行する。

* **エラーE1: 誘導タイムアウト**
  * **発生条件:** [STATE 2]において、おやつ排出から15秒経過しても犬が排泄地点から規定距離以上離れない場合。
  * **リカバリ:** ロボットは出動させない（接触リスク回避）。ステータスを[STATE 0]に戻し、Web側へ「誘導失敗」のアラート通知を送信する。

* **エラーE2: 移動中の犬の接近**
  * **発生条件:** [STATE 3] または [STATE 5]のロボット稼働中、カメラ映像内で「わんちゃん」と「ロボット」のバウンディングボックス間の距離が危険域（例: 1.0m以内）に入った場合。
  * **リカバリ:** 即座にロボットのモーター出力を強制カット（急停止）。犬が安全圏へ離脱するまで待機。1分以上状況が変わらない場合はフェイタルエラーとし、[STATE 0]へ遷移（ロボットはその場に放置）。

* **エラーE3: マーカー・ロスト (Marker Lost)**
  * **発生条件:** [STATE 3] または [STATE 5]の移動中、カメラの死角や光の反射によりロボットのArUcoマーカーを3秒以上連続で見失った場合。
  * **リカバリ:** 暴走を防ぐためロボットを即時停止。Web側へ「ナビゲーションエラー」を通知し、飼い主の遠隔操作（強制帰還コマンド等）を待つ。

## 4. Webシステムからの非同期介入 
飼い主が外出先からWebダッシュボード経由で介入する際の割り込みフロー。

* **キルスイッチ (緊急停止)**
  * **トリガー:** Web画面の「緊急停止」ボタン押下して、エッジPCが受信。
  * **挙動:** 現在のステートを強制的に破棄。ロボット、投下機構、おやつシューターへの全通信を遮断し、物理的な動作を一切停止させる。システム再起動まで[STATE: 停止]に固定。
* **強制帰還**
  * **トリガー:** ロボットが変な場所で停止している際など、Webから帰還指示を出す。
  * **挙動:** 現在のステートを破棄し、強制的に[STATE 5: 帰還]へ遷移させ、初期位置へ戻す試みを行う。
