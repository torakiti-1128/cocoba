# 03_FUNCTIONAL_REQUIREMENTS.md（機能要件）

## 1. 視覚・検知機能 (Vision)
* **FR-11 (ストリーミング受信):** ネットワークカメラからのRTSP映像をエッジPCで常時受信し、デコードできること。
* **FR-12 (AI物体識別):** YOLOv8nを用いて映像内から「犬」「排泄ポーズ」「排泄物」を識別できること。
* **FR-13 (座標変換):** 検知物のピクセル座標を、カメラの歪み補正を行った上で床面の2D絶対座標(X, Y)へ変換できること。
* **FR-14 (自己位置推定):** ロボット上のArUcoマーカーを検知し、ロボットの現在位置と向きをリアルタイムに把握できること。
* **FR-15 (環境適応):** 低照度時にカメラの赤外線(IR)暗視モードへ自動切り替えを行い、夜間でも検知を継続できること。

## 2. 誘導機能 (Decoy)
* **FR-21 (おやつ排出制御):** エッジPCからの指令を受信し、ESP32経由でおやつシューターを駆動できること。
* **FR-22 (排出量制御):** 1回の動作で規定量（数粒）のみを正確に排出し、過剰給餌を防止すること。

## 3. ロボット制御・投下機能 (Robot)
* **FR-31 (自律ナビゲーション):** ビジュアルサーボ制御により、目標座標の誤差5cm以内までロボットを自律移動させられること。
* **FR-32 (シールド投下):** 投下機構のサーボを制御し、スタックされたドームを1つずつ確実に落下させられること。
* **FR-33 (自動帰還):** タスク完了後、またはエラー時に、初期位置（充電ドック）へ自動で帰還できること。
* **FR-34 (ドーム残弾管理):** 物理的な残弾数をカウントし、WebUIへ同期できること。

## 4. 遠隔管理・Web連携機能 (Web/Cloud)
* **FR-41 (リアルタイム状態同期):** システムのステータス（待機/出動/エラー等）を、AWS IoT Core経由でWebUIへ即時反映できること。
* **FR-42 (エビデンス画像アップロード):** 排泄検知時、S3の署名付きURLを取得してスナップショット画像をセキュアにアップロードできること。
* **FR-43 (プッシュ通知):** 投下完了や異常発生時に、LINE Messaging APIを通じて飼い主へ通知を送信できること。
* **FR-44 (リモート・キルスイッチ):** WebUIからの緊急停止命令を受信し、全アクチュエータを即時停止できること。
* **FR-45 (遠隔帰還指示):** WebUIから強制的に「帰還モード」へ遷移させられること。
