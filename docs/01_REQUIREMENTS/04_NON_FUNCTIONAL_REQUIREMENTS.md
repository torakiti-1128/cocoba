# 04_NON_FUNCTIONAL_REQUIREMENTS.md（非機能要件）

## 1. 安全性・フェイルセーフ (Safety)
* **NFR-11 (近接停止):** ロボット稼働中、映像上で「わんちゃん」と「ロボット」の距離が1.0m以内に接近した場合、1秒以内にロボットを強制停止させること。
* **NFR-12 (誘導タイムアウト):** おやつ排出後、15秒経過しても犬が排泄地点から1.5m以上離れない場合は出動をキャンセルし、待機状態へ戻ること。
* **NFR-13 (通信断絶保護):** エッジPCとESP32間のMQTT通信が3秒以上途絶えた場合、マイコン側のWDT（ウォッチドッグタイマー）により全出力をゼロ（停止）にすること。
* **NFR-14 (物理キルスイッチ):** ロボット本体に、人手が直接全電源を遮断できる非常停止ボタンを搭載すること。
* **NFR-15 (電源分離):** モーター駆動系とマイコン制御系の電源を物理的に分離し、ノイズや電圧降下による誤動作を防止すること。

## 2. 性能・パフォーマンス (Performance)
* **NFR-21 (推論速度):** エッジPC上でYOLOv8nモデル（OpenVINO最適化済み）を用い、CPU負荷70%以下で10 FPS以上の推論速度を常時維持すること。
* **NFR-22 (操作遅延):** WebUIから緊急停止ボタンを押下し、ロボットが停止するまでのエンドツーエンドの遅延を2秒以内とすること。
* **NFR-23 (エッジ自律稼働):** クラウドとの接続が一時的に遮断された場合でも、ローカルネットワーク内の「検知〜投下」のコア機能を完結できること。
* **NFR-24 (低熱設計):** Intel N100のサーマルスロットリングを回避するため、長時間稼働時でもパッケージ温度を80度以下に抑える排熱設計を行うこと。

## 3. セキュリティ (Security)
* **NFR-31 (デバイス認証):** エッジPCは、個別のX.509証明書を用いてAWS IoT Coreへ認証・接続すること。
* **NFR-32 (通信暗号化):** 外部（クラウド/WebUI）との全通信は、TLS 1.2以上のプロトコルで暗号化すること。
* **NFR-33 (アクセス制御):** AWS Cognitoを利用し、認可されたユーザー（飼い主）のみがWebUIの操作・データの閲覧を行えること。
* **NFR-34 (データ保護):** S3上のスナップショット画像へのアクセスは、有効期限付きの署名付きURL（Presigned URL）に限定すること。

## 4. 運用性・保守性 (Operability)
* **NFR-41 (デイリーSOP):** 運用者による出発前・帰宅後の物理的なセットアップ（ドーム装填、清掃等）を、それぞれ3分以内で完了できる設計であること。
* **NFR-42 (自動復帰):** 停電等により電源が断絶した後、復旧時にエッジPCおよびデバイスが自動的に起動し、自律的に待機状態へ戻ること（Auto Power On）。
* **NFR-43 (耐環境性):** 湿気（排泄物付近）や、カリカリの油分によるセンサー・駆動部の汚れに耐えうる物理構造、または容易に清掃可能な構造とすること。
