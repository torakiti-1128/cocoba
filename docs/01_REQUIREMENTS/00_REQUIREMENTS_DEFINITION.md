# Cocoba: 要件定義書

## 1. 業務要件 
本システムは、14歳のシニア犬（ここちゃん）の排泄物踏み散らかしを防止し、運用者（飼い主）の清掃コストと精神的負担を削減するための自律・遠隔管理型IoTシステムである。

* **BR-01:** 帰宅後の排泄物関連の清掃時間を「毎日3分以内」に短縮する。
* **BR-02:** システム稼働による犬への接触事故・怪我を「完全にゼロ」とする。
* **BR-03:** 外出先からのステータス確認と緊急介入を可能とし、運用者の不安を解消する。

## 2. 機能要件
システムが必ず実装しなければならない具体的な機能。これらが一つでも欠ければリリース不可とする。

### 2.1 視覚・検知機能
* **FR-11:** ネットワークカメラのストリーミング映像(RTSP)をエッジPCで常時受信できること。
* **FR-12:** AIが映像内から「犬」「排泄ポーズ」「排泄物」を識別できること。
* **FR-13:** 検知した「排泄物」および「ロボット上のArUcoマーカー」のカメラ画像上のピクセル座標を、床面の2D絶対座標(X, Y)へ変換できること。

### 2.2 誘導機能 (Decoy)
* **FR-21:** エッジPCからのAPIリクエストを受信し、ESP32マイコン経由でおやつシューターを稼働できること。
* **FR-22:** 1回の稼働につき、少量の誘引餌を正確に排出できること（過剰給餌の防止）。

### 2.3 ロボット制御・投下機能
* **FR-31:** エッジPCが計算した目標座標へ、ベースロボットを自律移動（ビジュアルサーボ制御）できること。
* **FR-32:** 目標座標到達後、投下機構を稼働させ、シールド（ドーム）を「1つだけ」落下させることができること。
* **FR-33:** 投下完了後、ロボットが初期待機位置（または充電ドック）へ自動で帰還できること。

### 2.4 遠隔管理・Web連携機能
* **FR-41:** エッジPCからWebサーバーへ、現在のステータス（待機中/処理中/エラー等）およびドーム残弾数を定期同期できること。
* **FR-42:** 排泄物検知時に、その瞬間のスナップショット画像をWebサーバーへアップロードできること。
* **FR-43:** 投下完了時、またはエラー発生時に、飼い主のスマートフォンへプッシュ通知を送信できること。
* **FR-44:** Webダッシュボードから「緊急停止（キルスイッチ）」コマンドを発行し、エッジPCへ即時反映できること。

## 3. 非機能要件
システムの品質、性能、安全性に関する制約。現場の実用性を担保する。

### 3.1 安全性・フェイルセーフ
* **NFR-11 (近接停止):** ロボット稼働中、カメラ映像上でロボットと犬の距離が「1.0m以内」に接近した場合、いかなる状態でもロボットを1秒以内に強制停止させること。
* **NFR-12 (タイムアウト):** 誘導餌の排出後、15秒経過しても犬が排泄物から1.0m以上離れない場合は出動をキャンセルし、待機状態へ戻ること。
* **NFR-13 (通信断絶):** エッジPCとロボット間の通信が1秒以上途絶えた場合、ロボット側マイコンのフェイルセーフによりモーター出力をゼロにすること。

### 3.2 性能・パフォーマンス
* **NFR-21 (推論速度):** エッジPC上でのYOLO推論は、CPU負荷を70%以下に維持しつつ、最低 10FPS 以上で実行できること。
* **NFR-22 (キルスイッチ応答):** Webダッシュボードから緊急停止ボタンを押下してから、エッジPCがコマンドを受信しロボットを停止させるまでの遅延は「2秒以内」とすること。
  * ※この低レイテンシを実現するため、WebUI (Vercel) は AWS IoT Core への MQTT over WebSocket 直接接続を行い、中継の API Gateway を介さずにコマンドを送信する。

### 3.3 運用性・保守性
* **NFR-31 (日次SOP):** 運用者の「出発前セットアップ」および「帰宅後リカバリ」の物理的作業が、それぞれ3分以内で完了するハードウェア設計であること。
* **NFR-32 (耐熱・連続稼働):** エッジPCは24時間365日の稼働を前提とし、サーマルスロットリング（熱暴走による性能低下）を起こさない排熱性能を持つこと。

## 4. 前提条件と制約事項
* **CON-01:** システムはワンルームでの運用を前提とする。
* **CON-02:** クラウドシステムがダウンした場合でも、エッジPC内のローカルネットワーク通信のみで「検知〜投下」のコア機能は完結すること（エッジ自律の原則）。
