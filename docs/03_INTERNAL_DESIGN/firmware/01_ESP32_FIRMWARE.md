# 01_ESP32_FIRMWARE (ファームウェア・RTOS設計)

## 1. 概要
ロボットおよびシューターを制御する ESP32 のファームウェア設計を定義する。
物理的な安全性を担保するため、通信遅延による制御ブロックを防ぐ **FreeRTOS** を用いたマルチタスク設計と、ハードウェアタイマーによるフェイルセーフを実装する。

## 2. FreeRTOS タスク設計

ESP32はデュアルコア（Core 0, Core 1）を搭載している。ネットワーク処理と物理制御を完全に分離する。

### 2.1. Task 1: ネットワーク通信 (Core 0)
*   **役割:** Wi-Fi接続維持、MQTTメッセージの送受信。
*   **処理:**
    *   `cocoba/local/robot/move` などのトピックを購読。
    *   受信したメッセージをパースし、スレッドセーフなキュー（Queue）またはグローバル変数（ミューテックス保護）を介して制御タスクへ渡す。
    *   1秒に1回、生存確認（Heartbeat）を送信。
*   **優先度:** 中 (Priority 2)

### 2.2. Task 2: 物理制御 (Core 1)
*   **役割:** モーターのPWM制御、エンコーダの読み取り、サーボの駆動。
*   **処理:**
    *   通信タスクからの「目標速度 $(v, w)$」や「動作コマンド」を読み取る。
    *   10ms周期（100Hz）のタイマー割り込みでPID制御ループを回し、モータードライバへPWM信号を出力する。
    *   `delay()` 関数は絶対に使用しない（ノンブロッキング）。
*   **優先度:** 高 (Priority 5)

## 3. フェイルセーフ機構 (安全第一)

### 3.1. Watchdog Timer (WDT)
*   通信断絶時におけるロボットの暴走を防ぐ。
*   制御タスク (Task 2) は、通信タスク (Task 1) から新しいコマンドを受信するたびに「ソフトウェアタイマー」をリセットする。
*   **発動条件:** 最後にコマンドを受信してから **1.5秒** が経過した場合。
*   **発動アクション:** モーターのPWM出力を強制的に `0` にし、ロボットをその場で停止させる。

### 3.2. ハードウェアキルスイッチ（割り込み）
*   ESP32の特定のGPIOピン（例: GPIO 33）に、物理的な非常停止ボタンを接続する。
*   このピンにはハードウェア割り込み (ISR: Interrupt Service Routine) を設定する。
*   **ISRの動作:** 割り込みが発生した瞬間に、モーター駆動用のEN（Enable）ピンを `LOW` に落とし、OSの処理を介さずに物理的に電力を遮断する。

## 4. メモリ管理の厳格化
*   `String` クラスの使用を完全に禁止する（ヒープ断片化によるフリーズ防止）。
*   JSONのパースには `ArduinoJson` を使用し、事前に確保された静的メモリプール (`StaticJsonDocument`) のみを使用する。
*   `new` や `malloc` は `setup()` 関数内でのみ許可し、`loop()` や各タスク内での動的メモリ確保は行わない。
