# 01_VISION_ALGORITHM (視覚アルゴリズム設計)

## 1. 概要
本ドキュメントは、エッジPCのカメラ映像から「犬」「排泄物」「ロボット」を検出し、2次元画像座標（ピクセル）から実世界座標（メートル）への変換を行う一連のアルゴリズムを定義する。

## 2. 処理パイプライン

カメラからのRTSPストリームに対し、フレームごとに以下の処理を直列で実行する。
N100のCPUリソースを節約するため、処理は最大10FPSに制限（間引き）する。

```mermaid
graph TD
    A[RTSP Stream] --> B(フレーム抽出・リサイズ)
    B --> C{YOLOv8 推論}
    C -->|犬・排泄物| D[Bounding Box 解析]
    C -->|マーカー| E[ArUco 辞書解析]
    D --> F(Homography 変換)
    E --> F
    F --> G[実世界座標 (X, Y) 算出]
    G --> H[State Machine へイベント発行]
```

## 3. オブジェクト検出 (YOLO & ArUco)

### 3.1. 生体・物体検出 (YOLOv8)
*   **対象:** `dog` (犬), `poop` (排泄物)
*   **推論エンジン:** ONNX Runtime (CPU)
*   **モデル:** YOLOv8n (nano) を INT8 量子化して使用。
*   **座標抽出:** バウンディングボックスの下部中央（足元や接地地点）をオブジェクトの「ピクセル座標」として定義する。

### 3.2. ロボット追跡 (ArUco マーカー)
YOLOによる学習が不要で、高精度かつ高速に位置と向き（姿勢）を特定できるため、ロボット天板には OpenCV の **ArUcoマーカー** を貼付する。
*   **辞書:** `DICT_4X4_50`
*   **抽出情報:** マーカーの中心座標 $(P_x, P_y)$ および、4隅の座標から算出されるロボットの「向いている角度（Yaw）」。

## 4. 座標変換 (Homography)

斜め上から見下ろすカメラ映像（透視投影）のピクセル座標を、床面（真上から見下ろした2D平面）の実世界座標系に変換する。

### 4.1. キャリブレーション手順
1.  部屋の床の4隅（または既知の矩形）のピクセル座標を取得。
2.  現実世界の寸法（例: 2m x 2m）を入力。
3.  OpenCVの `cv2.findHomography()` を用いて変換行列 $H$ を算出。
4.  $H$ を `homography.json` としてエッジPC内に保存。

### 4.2. 座標変換の数式
ピクセル座標 $(u, v)$ を床面の物理座標 $(X, Y)$ へ変換する：
$$
\begin{bmatrix} x' \\ y' \\ w' \end{bmatrix} = H \begin{bmatrix} u \\ v \\ 1 \end{bmatrix}
$$
$X = x' / w'$ ,  $Y = y' / w'$

## 5. 距離計算と安全判定

ステートマシンに「犬との距離」を通知するためのロジック。
*   **ユークリッド距離:** 犬の座標 $D(X_d, Y_d)$ と、ロボットの座標 $R(X_r, Y_r)$ 間の距離。
    $$ L = \sqrt{(X_d - X_r)^2 + (Y_d - Y_r)^2} $$
*   **近接判定:** $L < 1.0$ (メートル) の場合、`PROXIMITY_ALERT` イベントをステートマシンへ即座に発行する。
