# 02_INTEGRATION_TEST (結合テスト計画)

## 1. エッジPC ⇔ ローカルデバイス (MQTT)

### 1.1. コマンド疎通
*   **テスト内容:** エッジPCから `cocoba/local/robot/move` をパブリッシュし、ESP32（実機またはシミュレータ）が受信してログ出力すること。
*   **期待結果:** レイテンシが 100ms 以内であること。

### 1.2. 生存確認 (Heartbeat)
*   **テスト内容:** ESP32の電源を遮断し、エッジPCのステートマシンが 3秒以内に `EMERGENCY` 状態へ遷移すること。
*   **期待結果:** WebUIへ「ロボットオフライン」の通知が飛ぶこと。

---

## 2. エッジPC ⇔ AWS クラウド (IoT Core / S3)

### 2.1. 状態ミラーリング
*   **テスト内容:** エッジPCの状態変更に伴い、DynamoDB の `SystemConfigTable` が自動的に更新されること。
*   **期待結果:** WebUI（または `awslocal`）から最新状態が確認できること。

### 2.2. スナップショット送信
*   **テスト内容:** エッジPCが S3 (LocalStack) へ画像をアップロードし、生成された署名付きURLでブラウザから閲覧できること。

---

## 3. Web UI ⇔ AWS ⇔ エッジPC

### 3.1. 緊急停止 (キルスイッチ)
*   **テスト内容:** WebUIの「緊急停止」スワイプを実行。
*   **期待結果:** IoT Core を経由してエッジPCが受信し、1秒以内にロボットへ停止命令が飛ぶこと。

### 3.2. 設定変更の即時反映
*   **テスト内容:** WebUIで「フレームレート」を 10fps から 5fps に変更。
*   **期待結果:** エッジPC内の推論ループの間隔が即座に 200ms へ変更されること。
