# 01_UNIT_TEST (単体テスト計画：再定義)

## 1. 推論レイヤー (Python / Inference)

### 1.1. Homography 変換行列の検証
*   **テストケース:** 
    *   4隅の対応点 $(u, v) \leftrightarrow (X, Y)$ を入力。
    *   中心点および画像端の点での変換誤差が $5mm$ 以内であることを確認。
    *   逆行列が存在しない（点が同一直線上にある）場合の例外処理。

### 1.2. 移動ベクトル算出 (Pure Pursuit)
*   **テストケース:** 
    *   ターゲットが真前、真横、真後にある場合の $(v, \omega)$ 指令値の算出。
    *   目標座標への到達距離 $e_d < 0.05m$ で $(0, 0)$ が返されること。
    *   最大速度制限 (Clipping) が $0.3m/s$ で機能していること。

### 1.3. 活動状態判定エンジン
*   **テストケース:**
    *   **ACTIVE:** 1秒ごとに $0.1m$ ずつ動く座標データを5個入力 -> `ACTIVE` 遷移。
    *   **SLEEP:** 同一座標を30個連続入力 -> `SLEEP` 遷移。
    *   **POOPING:** `poop_pose` クラスの信頼度 0.8 が 2秒継続 -> `POOPING` 遷移。

## 2. 制御レイヤー (C++ / Firmware)

### 2.1. 逆運動学 (Inverse Kinematics)
*   **テストケース:**
    *   $v=0.1, \omega=0$ -> 左右のデューティ比が一致。
    *   $v=0, \omega=1.0$ -> 左右のデューティ比が逆符号で絶対値一致（超信地旋回）。
    *   計算結果が PWM 範囲（0-255）を超えた場合の飽和処理。

### 2.2. Watchdog Timer (WDT) 
*   **テストケース:**
    *   `loop()` 内で MQTT 受信フラグを強制的に false に固定。
    *   1,500ms 経過時に `digitalWrite(MOTOR_EN, LOW)` が実行されることをロジックアナライザで確認。

## 3. クラウドレイヤー (Python / Lambda)

### 3.1. 統計計算エンジン
*   **テストケース:**
    *   **昨日比:** `ActivityLogs` から count=5 (昨日), count=3 (一昨日) をモック入力 -> `+2` を出力。
    *   **日付跨ぎ:** 23:59:59 と 00:00:01 のログが正しく別日に集計されること。

### 3.2. S3 署名付き URL 生成
*   **テストケース:**
    *   生成された URL が `Expires=300` (5分) であること。
    *   Content-Type が `image/jpeg` に制限されていること。
