# 01_UNIT_TEST (単体テスト計画)

## 1. エッジPC (Python)

### 1.1. 視覚アルゴリズム (`inference`)
*   **座標変換テスト:**
    *   既知のピクセル座標 $(u, v)$ を与え、Homography変換後の物理座標 $(X, Y)$ が計算値と一致するか（許容誤差 1cm以内）。
*   **距離計算テスト:**
    *   犬 $(1.0, 1.0)$ とロボット $(1.0, 2.0)$ の座標から、距離 $1.0m$ が算出されるか。
*   **静止判定テスト:**
    *   座標リストを入力し、5秒間で 0.1m しか動いていない場合に `SLEEP` 状態と判定されるか。

### 1.2. ステートマシン (`logic`)
*   **遷移トリガーテスト:**
    *   `STANDBY` 状態に `DETECT_POOP` イベントを入力した際、正しく `DETECTED` へ遷移するか。
*   **タイムアウトテスト:**
    *   `GUIDING` 状態から 15秒間イベントがない場合に `STANDBY`（またはエラー）へ戻るか。

## 2. バックエンド (AWS Lambda)

### 2.1. 統計集計ロジック
*   **昨日比計算テスト:**
    *   前日のウンチ回数 2回、一昨日のウンチ回数 1回 の時、`trend: "+1"` が算出されるか。
*   **APIバリデーション:**
    *   不正な `device_id` でリクエストした際に、`403 Forbidden` または `400 Bad Request` を返すか。

## 3. ファームウェア (C++)

### 3.1. モーター制御
*   **逆運動学テスト:**
    *   目標速度 $v=0.1, \omega=0$ のとき、左右のタイヤの回転指令値が同一かつ正の値になるか。
*   **フェイルセーフ:**
    *   WDT（ウォッチドッグタイマー）が 1.5秒で発動し、PWM出力が `0` になるか。

## 4. フロントエンド (Next.js)

### 4.1. UIコンポーネント
*   **配色テスト:**
    *   `systemState` が `EMERGENCY` の時、背景やステータス表示が「クリティカル赤」に変化するか。
