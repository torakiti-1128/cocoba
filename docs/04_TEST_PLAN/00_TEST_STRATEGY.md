# 00_TEST_STRATEGY (テスト戦略：再定義)

## 1. 目的と網羅基準
本ドキュメントは、Cocobaシステムの「物理的安全性」「AI検知精度」「低レイテンシ通信」を科学的・定量的に検証するための戦略を定義する。
テストの網羅基準は、`docs/01_REQUIREMENTS` に定義された全ての機能・非機能要件および KPI に直接紐付けられる。

## 2. 成功定義 (KPI) との紐付け
以下の目標数値をパスしない限り、本番稼働（ここちゃんへの適用）は許可されない。

| KPI項目 | 目標値 | 検証方法 |
| :--- | :--- | :--- |
| 排泄検知率 (Recall) | 98% 以上 | 1,000枚のテストデータセットによる評価 |
| 誤検知率 (Precision) | 95% 以上 | 24時間監視における空振り回数の計測 |
| 接触・衝突事故 | 0 件 | シナリオテストでの物理ダミーを用いた検証 |
| 通知遅延 (End-to-End) | 10秒以内 | 検知からスマホ通知受信までのタイムスタンプ差分 |
| ロボット停止遅延 | 1秒以内 | 接近検知からモーター出力ゼロまでの高速カメラ計測 |

## 3. テストフェーズと環境

### 3.1. ソフトウェア・シミュレーション (DEV)
*   **AI Inference:** 録画データ（昼・夜・夕方・電球色）を用いたオフライン推論テスト。
*   **Cloud Logic:** LocalStack および SAM Local による擬似クラウド環境。
*   **Frontend:** Jest + React Testing Library によるコンポーネント単体テスト。

### 3.2. ハードウェア・机上テスト (HIL: Hardware-in-the-Loop)
*   **ESP32 Logic:** モーターを接地させず、PWM波形をオシロスコープまたはロジックアナライザで確認。
*   **MQTT Stress:** 毎秒100メッセージを送信し、ESP32のバッファオーバーフローが発生しないか確認。

### 3.3. 実環境・模擬テスト (FIELD)
*   **Target:** 犬の代わりに「自律走行する小型ラジコン（犬役）」や「ぬいぐるみ」を使用。
*   **Poop:** 形状・色が異なる模擬排泄物（粘土等）を複数箇所に配置。

## 4. 異常系・境界値テストの重点項目
1.  **通信断絶:** Wi-Fi切断、MQTTブローカー停止時の自律停止 (WDT/LWT)。
2.  **電力不足:** バッテリー電圧降下時の挙動とクラウドへの警告送信。
3.  **環境変化:** 暗闇 (0 lux)、直射日光、床への映り込みによるAIの誤判定。
4.  **リソース枯渇:** N100のCPU負荷 100% 継続時のサーマルスロットリング発生下での動作維持。
