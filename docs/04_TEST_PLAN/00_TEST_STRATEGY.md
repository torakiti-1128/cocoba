# 00_TEST_STRATEGY (テスト戦略)

## 1. 目的
本ドキュメントは、Cocobaシステムの品質および安全性を保証するための全体的な検証方針を定義する。シニア犬の安全を最優先とするため、物理的な動作検証には慎重な多段階テストを適用する。

## 2. テストの階層

| 階層 | 対象 | 環境 | 主要ツール |
| :--- | :--- | :--- | :--- |
| **単体テスト** | 関数の計算ロジック、APIレスポンス | ローカル | pytest, jest, unittest |
| **結合テスト** | コンポーネント間通信 (MQTT/REST) | ローカル(Docker) | LocalStack, Mosquitto |
| **シナリオテスト** | 一連の状態遷移フロー (検知〜帰還) | 現場 (エッジPC) | 実機 + 模擬データ |
| **運用テスト** | 実稼働環境での連続稼働、安全性 | 自宅 (本番) | 実機 + ここちゃん |

## 3. テスト環境戦略

### 3.1. 仮想環境 (Simulation)
ハードウェアの摩耗や犬へのリスクを避けるため、可能な限りソフトウェア上で完結させる。
*   **AI推論:** 録画済みの動画ファイルを入力として流し、検知精度と座標変換を検証する。
*   **クラウド:** LocalStack を使用し、AWSアカウントなしで S3/DynamoDB 連携をテストする。

### 3.2. モック (Mocking)
*   **ロボット:** 実際のモーターを回す代わりに、MQTTで受け取った速度指令値をログ出力する「ダミーESP32」を使用する。
*   **天気API:** 外部APIを叩かず、固定のJSONを返すスタブを使用する。

## 4. 安全性検証の基本原則
物理的な移動を伴うテストにおいては、以下の「安全3原則」を遵守する。
1.  **物理キルスイッチの常時待機:** テスト実施者は常にロボットの1m以内に位置し、物理ボタンを即座に押せる状態にする。
2.  **机上テストの先行:** モーターを空転（タイヤを地面から浮かせる）させた状態で、指令値が正しいかを確認してから接地させる。
3.  **非生体ターゲット:** 運用テストの最終段階まで、本物の犬（ここちゃん）の代わりに「ぬいぐるみ」や「動くターゲット」を使用する。
