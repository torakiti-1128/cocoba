# 03_HARDWARE_SAFETY (ハードウェア安全・取り扱い規約)

## 1. 基本方針 (最優先事項)
本プロジェクトは、14歳のシニア犬（ここちゃん）が生活する空間で稼働する物理システムである。
いかなるソフトウェアの利便性や開発の効率化も「犬への安全性」と「火災等の物理的事故の防止」に優先してはならない。疑わしい挙動や配線がある場合は、即座にシステムの主電源を物理的に遮断すること。

## 2. 電源および配線に関する安全規約
電子部品の焼損（ショート）を防ぐための絶対ルール。

* **電源投入前のトリプルチェック:**
  ESP32やモータードライバに初めて電源を入れる前、および配線を変更した後は、必ず以下の3点を確認すること。
  1. VCC (プラス) と GND (マイナス) がショート（短絡）していないか、テスターの導通チェックで確認する。
  2. 5V系パーツ（モーター等）と3.3V系パーツ（ESP32ロジック等）の配線が混触していないか確認する。
  3. 電源の極性（プラス/マイナス）が逆になっていないか、目視で指差呼称する。
* **モーター電源の分離:**
  サーボモーター（MG996R等）の駆動電源は、絶対にESP32のVCCピンから直接取らないこと（ESP32のレギュレータが焼損するため）。必ず外部電源（別系統のACアダプタや降圧モジュール）から供給し、GNDのみをESP32と共通化すること。
* **ホットプラグの禁止:**
  通電状態（電源ON）のまま、ジャンパワイヤを抜き差ししたり、センサーを接続したりする行為（ホットプラグ）は厳禁とする。必ず電源をOFFにしてから作業を行う。

## 3. 稼働テスト時の安全規約
実機（ロボットやディスペンサー）を動かす際のルール。

* **隔離テストの原則:**
  モーターや投下機構の初めての稼働テスト（単体テスト）は、必ず犬（ここちゃん）がいない別室、または犬のケージ内、または人の目の届く環境で、絶対に接触防止が可能な環境で行うこと。
* **キルスイッチの常時確保:**
  ロボットの走行テストや、おやつシューターの稼働テスト中は、PCの画面上ではなく「物理的に電源ケーブル（またはバッテリー）を即座に引き抜ける状態」で待機すること。異常音や暴走の気配があれば、迷わず物理的に電源を落とす。
* **可動部のクリアランス確認:**
  サーボモーターが動く範囲（アームやツメ）に、配線ケーブルが巻き込まれないように、結束バンド等でケーブルを確実に固定（ルーティング）してからテストを行う。

## 4. AI・ソフトウェア連携時の安全規約
ソフトウェア側からハードウェアを制御する際のフェイルセーフ。

* **PWM値のハードリミット設定:**
  エッジPCからESP32へモーターの回転指示を出す際、プログラム（C++側）で必ず「可動域の限界値（例：0度〜180度）」をハードコードで制限する。AIが異常な値を送信してきても、モーターが振り切れて機構が破壊されるのを防ぐ。
* **通信途絶時の自動停止:**
  ESP32側には、エッジPCとの通信（MQTT等）が1秒以上途絶えた場合、自律的にすべてのモーター出力をゼロ（停止）にするウォッチドッグタイマーを必ず実装する。これにより、PCがフリーズしてもロボットが暴走し続ける事故を防ぐ。

## 5. 運用時の環境安全規約
* **放熱の確保:** エッジPCやESP32を密閉ケースに入れる場合は、必ず排熱用のスリットや冷却ファンを設け、サーマルスロットリング（熱暴走）や発火のリスクを排除する。
* **水濡れ厳禁:** ここちゃんの飲み水や排泄物が、基板や電源タップにかからない位置にシステムを配置する。
