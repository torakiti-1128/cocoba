# PROGRESS_LOG_RULE (進捗ログ記述規約)

## 1. 基本方針
`07_PROGRESS_LOGS` 配下のファイルは、単なる開発日記ではない。AIが過去の文脈を喪失（コンテキストドロップ）した際に、プロジェクトの「現在地」と「直近の技術的決定事項」を即座に再ロードするための「AIの外部記憶装置（セーブデータ）」である。
最大の目的は「ユーザーの指示（プロンプト）」と「その結果（AIの出力）」をセットで記録し、AIの思考プロセスとソースコードの変更履歴を完全にトレース可能にすることである。後任のAI（別のペルソナ）が読んでも一切迷わない解像度でログを残す義務を負う。

## 2. ログ作成のタイミング
以下のいずれかの条件を満たした際、AIは自律的にログの出力を提案、またはユーザーの指示により出力しなければならない。

* 1つのタスクが完了したとき（タスク＝ユーザーの指示と、その実行結果のセット）。
* 日を跨ぐなど、開発セッションを一時中断・終了するとき。

※注意: AIのハルシネーションや不具合による後戻り作業、およびインシデントの記録は、本ディレクトリではなく `docs/08_INCIDENT_REPORTS/INCIDENT_REPORT_RULE.md` に従って別ディレクトリに分離して記録すること。

## 3. ファイル命名規則
ログファイルは、OS上で時系列順に自然に並び、かつ中身が推測できる厳格な命名規則とする。

**基本的な形式: `YYYYMMDDHHMM_summary.md` (分単位のタイムスタンプ)**
* 例: `202603161040_yolo_inference.md`
* 例: `202603171820_esp32_pwm_setup.md`

**中断・終了時の形式: `YYYYMMDDHHMM_session_end_summary.md`**
* 例: `202603161900_session_end_yolo_inference.md`

## 4. ログ記述フォーマット
AIが進捗ログを生成する際は、以下のMarkdownテンプレートに完全に準拠すること。項目の省略は一切許されない。

---
### 【AI向け出力テンプレート】
---

# 進捗ログ: 【ここにサマリーを1行で記載】

* **タイムスタンプ:** YYYY-MM-DD HH:MM
* **関連Issue:** #XXX
* **担当AI Skill:** 【例: エッジAI推論専門家】

## 1. ユーザーの指示（プロンプト）
* 【ユーザーから与えられた指示の要約、または重要な制約事項を記載】
* 【例: 「Issue #12に基づき、YOLOv8の推論パイプラインを実装せよ。ただしFPSは落とし、CPU負荷を優先すること」】

## 2. AIの実行結果（完了した作業）
* 【上記指示に対し、AIが具体的に何を実装・解決したかを箇条書きで記載】
* 【例: Intel N100環境でのYOLOv8推論パイプラインを構築。0.5秒に1回の推論（Drop処理）を行うFPS制限ロジックを実装完了】

## 3. 変更・新規作成した主要ファイル
* `src/【ファイルパス】`: 【追加・修正内容の端的な説明】
* `src/【テストファイルパス】`: 【TDDに基づくテスト実装の証明】

## 4. 重要な技術的決定・制約事項
* 【なぜその設計にしたのか、後続のAIや人間が「勝手に変えてはいけないルール」は何かを記載】

## 5. 次期タスク・引き継ぎ事項
* 【次に着手すべきIssue、または別の専門家（Skill）へ引き継ぐべき懸念点・バトンタッチの内容】

---
